# 首页综合仪表盘设计方案

## 概述

将首页从简单的信息展示升级为综合工作台，以通知为中心，整合任务统计，提供快捷操作入口。

## 页面结构

```
┌─────────────────────────────────────────────────────────────┐
│  时间问候语 + 用户头像 + Token 配额入口                       │
├─────────────────────────────────────────────────────────────┤
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐             │
│  │  时间   │ │ Token  │ │ 进行中 │ │ 未读   │             │
│  │  14:32 │ │ 2,850  │ │   3    │ │   5    │             │
│  └────────┘ └────────┘ └────────┘ └────────┘             │
├─────────────────────────────────┬───────────────────────────┤
│                                 │  任务小卡片               │
│     通知列表                     │  ┌─────┬─────┬─────┐    │
│                                 │  │  3  │  5  │ 12  │    │
│  🔔 新任务分配给你    [已读]     │  │进行中│ 待办 │已完成│    │
│  ⚠️ Blog 审核通过    [已读]     │  └─────┴─────┴─────┘    │
│  📧 系统更新通知      [已读]     ├───────────────────────────┤
│                                 │  快捷操作                 │
│                                 │  • 新建任务               │
│                                 │  • 查看全部通知           │
│                                 │  • 前往任务中心           │
└─────────────────────────────────┴───────────────────────────┘
```

## 布局比例

- 顶部统计区：4 列等宽
- 下方内容区：通知列表 60% / 右侧面板 40%

## 数据来源

| 组件 | 数据源 | 字段 |
|------|--------|------|
| Token 配额 | MiniMaxTokenService | `getMainModelRemain()` |
| 任务统计 | TaskService | `tasks.length` 按状态过滤 |
| 未读通知 | NotificationService | `unreadCount` |
| 通知列表 | NotificationService | `notifications[]` 取未读 |
| 任务列表 | TaskService | `tasks[]` 按状态过滤 |

## 组件设计

### 1. 统计卡片（StatsCard）

保留现有的 4 个统计卡片：
- 时间卡片 - 实时更新
- Token 配额卡片
- 进行中任务数卡片（点击跳转 /tasks）
- 未读通知数卡片（点击跳转 /notifications）

### 2. 通知列表（NotificationList）

- 垂直列表，每条通知包含：
  - 类型图标（任务/博客/系统）
  - 通知标题
  - 时间描述（相对时间：2分钟前、1小时前）
  - 已读/删除操作按钮
- 已读通知置灰显示
- 空状态：显示"暂无未读通知"

### 3. 任务小卡片（TaskStatsCard）

- 3 列布局：进行中 / 待办 / 已完成
- 每列显示数量和标签
- 点击数字跳转 /tasks 路由

### 4. 快捷操作区（QuickActions）

- 新建任务 → /tasks/new 或打开新建任务 Modal
- 查看全部通知 → /notifications
- 前往任务中心 → /tasks

## 实现要点

1. 使用现有 `@rabjs/react` Service 模式获取数据
2. 通知列表仅展示未读通知，已读数据不展示
3. 实时更新：Token 配额、通知数量使用定时刷新
4. 响应式：小屏幕下右侧面板移至通知列表下方

## 优先级

1. 添加通知列表组件
2. 添加任务统计卡片
3. 添加快捷操作区
4. 调整现有布局比例
