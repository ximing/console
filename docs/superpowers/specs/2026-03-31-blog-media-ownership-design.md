# Blog Media Ownership Design

## Problem

Currently `GET /api/v1/blogs/media/url` validates access by checking if the path contains `userId`:

```typescript
if (!pathParts.includes(userDto.id)) {
  return ResponseUtility.error(ErrorCode.FORBIDDEN, 'Access denied');
}
```

But S3 paths are `{userId}/{filename}`, so anyone can guess another user's path and access their images. No ownership record exists in the database.

## Solution

Create a `blog_media` table to track media ownership, enabling proper authorization checks.

## Data Model

### New Table: `blog_media`

| Column | Type | Description |
|--------|------|-------------|
| id | varchar(24) | Primary key, using `generateId()` with `m` prefix |
| blogId | varchar(191) | FK to blogs.id |
| path | varchar(500) | S3 object key |
| filename | varchar(255) | Original filename |
| size | int | File size in bytes |
| type | varchar(20) | Media type: 'image', 'audio', 'video' |
| width | int | Image width (optional) |
| height | int | Image height (optional) |
| createdAt | timestamp | Creation time |

Indexes:
- `blogId_idx` on (blogId)
- `path_idx` on (path) - unique

### S3 Path Format

Change from `{userId}/{filename}` to `{userId}/{blogId}/{filename}`

Example: `dev/console/u123abc/m456def/image.png`

## API Changes

### Upload: `POST /api/v1/blogs/media/upload`

**Request**: Add `blogId` form field

**Response**: Unchanged (still returns path, filename, size, type, width, height)

**New behavior**:
1. Validate blogId exists and belongs to user
2. Upload to S3 with path `{userId}/{blogId}/{filename}`
3. Insert record into `blog_media` table

### Get URL: `GET /api/v1/blogs/media/url`

**Request**: Unchanged (path query param)

**Response**: Unchanged

**New authorization flow**:
1. Parse path to get blogId
2. Query `blog_media` by path to get blogId
3. Query `blogs` to get status and userId
4. If `blog.status === 'published'` → return presigned URL directly
5. If `blog.status === 'draft'` → check `blog.userId === currentUser.id`
6. Deny if neither condition is met
