# Token 配额展示改进设计

## 概述

改进首页 Token 配额展示，从单一数字升级为显示所有 11 个模型的详细配额信息。

## 布局调整

### 调整前 Stats Grid（4 栏）
```
[Time Card] [Token Card] [Task Card] [Notification Card]
```

### 调整后布局
```
[Greeting 区域: 欢迎语 + 时间 + 头像]
[Token 独占一行: 主卡片(40%) + 小卡片网格(60%)]
[双栏: 通知列表(60%) | QuickActions(40%)]
```

**变更：**
- Time Card 移除，时间整合到 Greeting 区域
- Task Card 移除（与通知列表重复）
- Notification Card 移除，保留通知列表（更大面积）
- Token 卡片独占一行，扩展展示详情

## Token 行设计

### 主卡片（左侧，约 40% 宽度）

```
┌─────────────────────────────────────┐
│ MiniMax-M*                          │
│                                     │
│ 3,230                    ← 大字数字 │
│                                     │
│ [████████████░░░░░░░░░░░░░] 28%     │
│                                     │
│ 本周: 43,540 / 45,000    剩余 1小时  │
└─────────────────────────────────────┘
```

- **模型名称**：显示 `model_name`
- **大字数字**：剩余配额 `current_interval_total_count - current_interval_usage_count`
- **进度条**：使用量 / 总量，宽度撑满卡片
- **本周配额**：次要信息，小字显示
- **时间剩余**：`formatRemainsTime()` 显示如"剩余 2小时"

### 小卡片网格（右侧，约 60% 宽度）

```
┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐
│    │ │    │ │    │ │    │ │    │  ← Row 1 (5个)
└────┘ └────┘ └────┘ └────┘ └────┘
┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐
│    │ │    │ │    │ │    │ │    │  ← Row 2 (5个)
└────┘ └────┘ └────┘ └────┘ └────┘
┌────┐
│    │  ← Row 3 (1个)
└────┘
```

- 2行 x 5列 + 1个 = 11个模型
- 每个小卡片：模型名称 + 剩余数量 + 进度条
- 卡片高度：60px
- 间距：8px

### 小卡片结构

```
┌──────────────────┐
│ speech-hd         │
│ 19,000  [████]   │
└──────────────────┘
```

- 紧凑单行布局
- 模型名称（小字）
- 剩余数量（大字）
- 进度条（细条）

## 颜色策略

### 进度条颜色

| 剩余比例 | 颜色 | Tailwind class |
|---------|------|---------------|
| > 30% | 绿色 | `bg-green-500` |
| 10% - 30% | 黄色 | `bg-yellow-500` |
| < 10% | 红色 | `bg-red-500` |

### 卡片状态

- 剩余为 0 时：卡片 `opacity-50`，文字变灰
- `remains_time` < 1小时：时间显示红色
