# Wiki 风格博客布局设计方案

## 概述

将 `apps/web/src/pages/blogs` 页面重构为 Notion 风格的 Wiki 布局，提供左侧目录树 + 右侧动态内容的交互体验。

## 整体布局

```
┌─────────────────────────────────────────────────────────────────┐
│  整体页面（100% 高度）                                            │
├──────────────┬──────────────────────────────────────────────────┤
│              │                                                   │
│  左侧边栏     │   右侧内容区                                       │
│  (260px)     │   (flex-1)                                       │
│              │                                                   │
│  搜索图标     │   动态内容区：                                    │
│  目录树       │   - 未选择目录 → 最近博客列表                     │
│  新建按钮     │   - 选择目录 → 该目录下所有页面列表                │
│              │   - 选择页面 → 页面预览（只读）                     │
└──────────────┴──────────────────────────────────────────────────┘
```

## 左侧边栏

### 顶部区域
- **搜索图标按钮**：点击后打开搜索弹窗

### 目录树区域
- **"全部博客"节点**：点击显示最近博客列表
- **树形目录结构**：支持展开/折叠
- **右键菜单**：
  - 目录节点：新建博客、新建子目录、重命名、删除
  - 页面节点：在编辑器中打开、删除、移动到、复制链接

### 底部区域
- **新建目录按钮**
- **新建博客按钮**

## 右侧内容区

### 三种状态

#### 1. 初始状态（最近博客）
- 标题："最近"
- 页面卡片列表（无分页，按更新时间排序）
- 点击卡片进入预览

#### 2. 目录选中状态
- 标题：目录名称
- 返回按钮（返回"全部博客"/最近）
- 页面卡片列表（无分页）
- 点击卡片进入预览

#### 3. 页面预览状态（只读）
- 顶部：页面标题 + 面包屑（目录路径）
- 底部工具栏：编辑按钮（跳转编辑器）、返回列表按钮
- 内容区：博客富文本内容（只读渲染）

## 搜索弹窗

- **触发**：点击侧边栏顶部搜索图标
- **位置**：居中 Modal（宽度 560px）
- **交互**：
  - 自动聚焦搜索输入框
  - 实时搜索（debounce 300ms）
  - 显示匹配博客列表
  - 点击结果：关闭弹窗 + 进入预览 + 目录树定位

## 右键菜单

### 目录节点
| 操作 | 说明 |
|------|------|
| 新建博客 | 在该目录下创建新博客 |
| 新建子目录 | 在该目录下创建子目录 |
| 重命名 | 弹出输入框重命名目录 |
| 删除 | 确认后删除目录 |

### 页面节点
| 操作 | 说明 |
|------|------|
| 在编辑器中打开 | 跳转到 Tiptap 编辑器 |
| 删除 | 确认后删除博客 |
| 移动到 | 打开目录选择器 |
| 复制链接 | 复制博客链接到剪贴板 |

## 数据流

1. **目录树操作** → DirectoryService
2. **博客列表加载** → BlogService.loadBlogs() 无分页参数
3. **博客预览** → BlogService.loadBlog(id) 获取完整内容
4. **搜索** → blogApi.searchBlogs(query) 返回匹配列表

```
│   │   ├── index.tsx           # 内容区容器

```

## 路由

现有路由保持不变：
- `/blogs` - 博客列表页面
- `/blogs/:id/editor` - 博客编辑器

预览模式在页面内切换，不涉及路由变化。

## 实现顺序

1. 重构 sidebar 组件结构
2. 实现 content 区域三种状态
3. 实现搜索弹窗
4. 添加右键菜单功能
5. 移除 TagFilter 和 StatusFilter（不在需要）
6. 移除分页组件
7. 清理旧代码
